# Contributing

Thanks for your interest in contributing. Please open an issue to discuss larger changes before submitting a PR. For bug reports and small improvements, open an issue with reproduction steps and a minimal sample if possible.

Guidelines:
- Follow the project's code style and run `npm run lint` before submitting.
- Keep changes focused and document behavior changes in the PR description.
- For content or copy changes, edit the relevant Markdown files and submit a PR.

This file intentionally omits external maintainer or sponsorship links.
